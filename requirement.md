# 智能医疗箱管理系统（IMMS）
## 项目任务重规划与人员分工说明

## 一、系统整体说明（简述）

本系统为智能医疗箱管理系统（IMMS），用于实现家庭用药信息管理、用药提醒以及基础健康信息展示。
系统采用 **模块化、分层设计**，通过统一的数据模型和服务接口，实现功能的增量式开发与集成。

系统不追求复杂算法，重点体现：

- 功能完整性
- 模块解耦
- 软件工程规范性

## 二、系统总体架构划分

系统采用典型三层结构：

- 表现层（Controller / 前端）
- 业务层（Service）
- 数据层（Entity / Repository）

在此基础上，将系统功能划分为一个基础平台模块 + 三个增量功能模块。

## 三、基础平台模块（由 1 人统一完成）

**说明**：该部分作为系统公共基础，仅由一人完成，其他模块仅通过接口调用，不修改其内部实现。

### 1. 用户与身份基础模块

- 用户基本信息管理
- 家庭成员信息管理
- 用户与家庭成员的关联关系

**主要数据对象**：
- `User`
- `FamilyMember`

### 2. 药品基础信息模块

- 药品基本信息管理
- 药品分类管理
- 药品库存基础信息

**主要数据对象**：
- `Drug`
- `DrugCategory`

### 3. 用药提醒基础框架

- 提醒信息的统一数据结构
- 提醒时间与状态管理
- 提供提醒相关的基础服务接口

**主要数据对象**：
- `Reminder`

**注**：该模块只定义"提醒是什么"，不涉及具体提醒方式的实现。

### 4. 系统公共支撑

- 数据库配置
- 基础 CRUD 接口
- 项目整体结构搭建

## 四、增量功能模块划分（各成员独立完成）

各功能模块相互独立，仅依赖基础平台提供的数据和接口，模块之间不存在直接耦合。

### 模块 A：用药提醒功能增强模块

**负责内容**：
- 定时用药提醒
- 提醒触发与状态更新
- 提醒信息展示

**依赖模块**：
- 基础平台中的 Reminder 服务

**开发边界**：
- 不修改用户与药品的数据结构
- 仅扩展提醒相关业务逻辑

### 模块 B：药品信息管理与查询模块

**负责内容**：
- 药品信息查询
- 药品信息展示
- 药品基础知识说明（静态或简单数据）

**依赖模块**：
- 药品基础信息模块

**开发边界**：
- 不修改药品表结构
- 仅进行数据读取与展示

### 模块 C：用户健康与信息展示模块

**负责内容**：
- 用户个人健康信息展示
- 家庭成员信息展示
- 基础统计信息展示（如用药记录数量）

**依赖模块**：
- 用户与家庭成员模块

**开发边界**：
- 不涉及用药提醒逻辑
- 不修改核心数据模型

## 五、模块集成与协作方式说明

- 各模块通过 Service 接口进行集成
- 模块合并时以"新增功能模块"为主
- 不修改已有模块代码，避免冲突
- 系统最终功能由各模块组合形成

该方式支持并行开发，降低模块间耦合度，便于功能扩展与维护。
